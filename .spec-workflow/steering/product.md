# Product Steering

## プロダクト概要

bookmark-app-mock は、ブックマーク管理アプリを題材にした学習用モックプロジェクトである。  
すぐモックを作り始めるための最小テンプレとして、UI / 導線 / 体験の仮説確認に絞って検証する。

## 目的

- 本開発前の技術選定の精度を高める
- monorepo-template の刷新につなげる
- BE や API 周りの知識を深め、開発手法を身に着ける

## 利用前提

- 想定ユーザーは置かず、個人の学習目的と本開発前のモックアップ検証を前提にする

## 作る範囲

### 画面

- ユーザー登録
- コレクション管理
- ブックマーク一覧 / 追加 / 削除
- タグ / カテゴリ管理
- 検索 / フィルタ / ソート

### 機能

- ユーザー登録
- コレクション作成（親 / 子）
- ブックマーク追加（URL 登録）
- タグ / カテゴリ設定
- 一覧表示
- ピン止め
- 検索 / フィルタ / ソート
- 削除

## 使用想定技術

- `pnpm workspace` モノレポ
- TypeScript
- Hono（API）
- Kysely
- Atlas（schema + migration）
- PostgreSQL（Docker）
- zod
- Web は Next.js または TanStack 系（未確定、最小 UI）

## やらないこと

- ブックマークのタイトル取得
- OGP 取得
- 共有機能
- import / export
- ブラウザ拡張
- ごみ箱機能
- 3階層以上のコレクション
- UI の作り込み

## 成功指標

- 本開発前に技術構成の妥当性を検証し、改善点を明確にできる
- Monorepo のディレクトリ構成と責務分割の精度を高められる
- 仕様駆動開発の手順で実装を進められる
- DB 構築と API 開発（Kysely / Hono）への知見を深められる
- pencli.dev を使ったデザイン構築を試行できる
- beautiful-mermaid を使った図作成フローを確立できる

## 完了条件

- 以下の最低限項目をモックとして操作確認できる
- コレクション（ブックマーク格納フォルダ）の親子階層（2階層まで）を操作できる
- ブックマーク追加（URL 登録と URL 表示のみ）と削除を操作できる
- ブックマークのピン止め機能を操作できる
- ユーザー登録（ユーザー ID、ユーザー名、パスワード、アイコン）を操作できる
- タグとカテゴリを操作できる
- 検索 / フィルタ / ソートを操作できる

## プロダクト原則

- 最小で動く実装を先に作る
- 変更理由を説明できる単位で段階的に改善する
- 学習しやすさを最優先し、次に変更しやすさを優先する
- UI より API/DB/型の理解を優先する
