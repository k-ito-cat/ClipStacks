# docs 運用ルール

## 目的

このファイルは、`docs/` と `README.md` の同期に関するコアルールだけを定義する。

## コアルール

- 実装と `docs/` / `README.md` の乖離を放置しない
- 更新が必要そうな場合は後回しにせず、その場で同期要否を判断する
- 局所修正で終わらせず、変更の波及先を確認する
- docs 同期の詳細手順は `docs-sync-workflow` Skill を正本とする
- デザイン作成や `docs/design/` の整理は `design-workflow` Skill を正本とする
- ドキュメント更新は、対象、理由、更新方針を明示し、ユーザーの許可を得てから行う
